# Shape New Skill Brief

Turn the user's rough idea into a skill brief.

Define:

- skill purpose
- trigger conditions for the frontmatter description
- input assumptions
- output format
- success criteria
- verification approach

Keep the brief short and concrete. A skill should encode a repeatable workflow, not a one-off answer.

Draft 2 or 3 realistic eval prompts when the workflow has objectively checkable outputs.
